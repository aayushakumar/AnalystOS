# Executive Summary Guidelines

Every analytical response should be structured for a busy decision-maker. Lead with the answer, support with evidence, and close with caveats. Never bury the headline.

---

## Answer Structure

### 1. Key Finding (Lead)

State the single most important takeaway in one sentence. Use a concrete number.

> **Revenue grew 12% month-over-month to $2.4M in February 2025**, driven primarily by a 23% increase in the Electronics category.

**Rules:**
- Always include the actual metric value, not just the direction ("up", "down").
- Always include the time period.
- If there's a clear driver, mention it immediately.

### 2. Supporting Detail

Provide 2–4 bullet points of additional context that support or nuance the key finding.

> - Electronics revenue: $890K (+23% MoM), boosted by the Presidents' Day sale.
> - Apparel revenue: $410K (-5% MoM), consistent with seasonal patterns.
> - New customer orders accounted for 34% of total revenue, up from 28% last month.

**Rules:**
- Be specific — use numbers, percentages, and named segments.
- Order by importance or magnitude of impact.
- Each bullet should add new information, not restate the headline.

### 3. Caveats and Limitations

Disclose anything that qualifies the finding. This builds trust.

> - February had 28 days vs. January's 31 — daily revenue is a fairer comparison (+3% per day).
> - Returns data for February is not yet finalized; net revenue may adjust downward by ~2%.

## Confidence Framing

Always communicate how confident the answer is. Use one of these levels:

### High Confidence
Use when data is complete and the metric definition is unambiguous.

> **High confidence** — based on complete data for the requested period with a well-defined metric.

### Moderate Confidence
Use when there are minor gaps, assumptions, or the metric required interpretation.

> **Moderate confidence** — the data covers the requested period, but [specific caveat, e.g., "returns data is preliminary" or "we assumed 'active' means at least one login in 30 days"].

### Low Confidence
Use when data is significantly incomplete, the metric is non-standard, or major assumptions were made.

> **Low confidence** — [specific reason, e.g., "marketing spend data is only available through October, so Q4 CAC is estimated"].

**Rules:**
- Never say "high confidence" when you've made assumptions — that's misleading.
- Never say "low confidence" and then present the number as if it were authoritative.
- If confidence is low, proactively suggest what additional data would improve it.

## Limitation Wording Templates

Use these templates when communicating data limitations. Be direct but not alarming.

### Incomplete Data
> Note: **{data_source}** data is available through **{date}**. Results for **{incomplete_period}** are partial and may change.

### Proxy Metric
> We don't have a direct measure of **{ideal_metric}**, so I've used **{proxy_metric}** as an approximation. The two typically track closely but can diverge when **{divergence_condition}**.

### Assumption Made
> I assumed **{assumption}** because **{reason}**. If that doesn't match your intent, let me know and I'll adjust.

### Excluded Data
> I excluded **{what_was_excluded}** from this analysis because **{reason}**. Including it would change the result by approximately **{impact}**.

## Communicating Uncertainty

### Do
- Quantify uncertainty when possible: "Revenue is between $2.3M and $2.5M depending on returns."
- Use ranges instead of false precision: "approximately 12%" rather than "12.347%".
- Distinguish between data uncertainty and definitional uncertainty: "We're confident in the number but the metric definition may not match your intent."

### Don't
- Don't use hedge words that undermine the answer without adding information ("maybe", "perhaps", "it seems like").
- Don't present a caveat and then ignore it — if the caveat matters, adjust the answer or the confidence level.
- Don't overload with caveats — pick the 1–2 that actually matter.

## Insight Bullet Rules

When writing bullet points in the supporting detail section, follow these rules:

| Rule | Bad Example | Good Example |
|---|---|---|
| Be specific, use numbers | "Revenue went up" | "Revenue increased 12% to $2.4M" |
| Name the segment | "One category grew" | "Electronics grew 23% to $890K" |
| Include the time context | "Conversion is 3.2%" | "Conversion was 3.2% last week, up from 2.8%" |
| Avoid vague qualifiers | "Revenue was quite strong" | "Revenue was $2.4M, the highest since October" |
| State direction and magnitude | "Customer growth changed" | "New customers grew 15% MoM to 4,200" |
| Compare to a benchmark | "Churn was 8%" | "Churn was 8%, above our 5% target" |

---

## Response Length Guidelines

- **Simple factual question** (e.g., "What was revenue last month?"): 1–2 sentences + the number. No bullets needed.
- **Trend or comparison question**: Key finding + 2–3 supporting bullets + 1 caveat if relevant.
- **Deep-dive or multi-dimensional question**: Key finding + 3–5 supporting bullets + caveats + suggest follow-up questions.

Never pad a simple answer with unnecessary context. Respect the reader's time.
